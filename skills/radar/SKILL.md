# Meta-Cognition Radar

Source: Justin Sung, "The Metacognition Radar." The agent is the **radar-trainer**: install the instrument, run the A/P drill, log telemetry, hold the timeline. The user does the thinking. **Never learn for the user** — no summaries-as-shortcut.

**Core claim:** The study strategy barely matters. What matters is the cognitive state during it. Two people, identical technique — one learns, one wastes 90% of time. The difference is metacognition. **Perceived mental effort is the only visible signal** — high effort ≈ active, easy/drowsy ≈ passive. The radar detects passivity; building it takes ~1 month. Detection before correction.

## Modes

| Mode | Active? | What it adds |
|---|---|---|
| **1 — Core (radar)** | Always | A/P Sheet, effort signal, detection timeline. The engine. |
| **2 — Deep Work perimeter** | Optional | Phone away, notifications off, block scheduling. Guards against "quiet ≠ deep." Activate when user reports distraction. |
| **3 — Berean check** | Optional | For Scripture/policy/philosophy: test claims actively against worldview, don't absorb passively. Activate when material carries moral/teleological weight. |

**Mode 1 only = identical to the original A/P sheet approach.** Modes 2–3 are bolt-ons.

## Core Concepts

| Concept | Rule |
|---|---|
| **Metacognition** | Awareness of your own thoughts. Not a study technique — a layer *above* cognition. |
| **Mental effort** | The ONLY signal. Deep thinking feels effortful; passive feels easy (brain checks out, daydreams). Route every check back to: "How hard was your brain working just then?" |
| **Active vs. Passive** | Active: "Read as if teaching skeptical experts who'll interrogate you." Passive: read start-to-finish trying to understand. Low effort, zero retention. |
| **Pattern ceiling** | Brain develops preferred patterns. You hit a wall when a topic demands a different kind of thinking. Not a talent ceiling — a visibility problem the radar fixes. |
| **Detection before correction** | ~1 month to build the radar (catch every passivity slip). Active strategies take 1–2 weeks after detection is reliable. |

## The A/P Sheet (The Engine)

Each session:

1. **If Mode 2 active:** confirm perimeter — phone away, notifications off, single task, 45–90 min block. No perimeter → A/P data is noise (can't distinguish distraction from passivity).
2. User studies in **Active** mode — the teaching frame.
3. Every time they catch a drift into passive (daydreaming, drowsy, re-reading), mark **P** with 2–4 word cause note. Re-orient to Active. Continue.
4. Afterward, agent logs metrics:
   - **Passive ratio** — trending down over weeks
   - **Detection latency** — how fast they caught it. The real metric; should drop session over session.
5. Agent interrogates P-notes for patterns: time-of-day? Material type? Specific concepts that induce autopilot?

## Audit & Compass (After Detection is Reliable)

Once ~80%+ catch rate (~1 month at 10+ hrs/week), supply active moves:

| Trigger | Move |
|---|---|
| Drowsy on reading | Switch to retrieval — teach to empty room out loud |
| Autopilot on familiar concept | Force novel reformulation — rewrite with new structure |
| Passive re-reading | Self-test from blank page before re-reading |
| Abstract material | Mind-map / diagram from memory |

## Timeline

| Phase | Duration (~10+ hrs/wk) | Goal |
|---|---|---|
| Build the radar | ~1 month | Detect every passivity slip reliably |
| Apply the compass | 1–2 weeks | Active switching becomes routine |

## On-Ramp

1. **Today:** one A/P session, 60–90 min. Just observe. How often P?
2. **This week:** 3–5 sessions. Goal: catch drift *faster* each time.
3. **Month 1:** keep radar running. At ~80%+ catch rate, start active strategies.

## Output

Maintain one radar log per learning target in Obsidian (`skill_view('note-taking/obsidian')`). Template: `templates/radar-log.md`. Frontmatter `stage:` tracks progress (calibrate / detecting / auditing / competent). Log: date, block length, passive ratio, detection latency, recurring triggers.

## Pitfalls

| Problem | Fix |
|---|---|
| Quiet-hour trap: undistracted ≠ deep | A/P telemetry must prove the block was deep, not merely quiet |
| Detection before correction violated | Don't chase active strategies in week 1. Radar first. |
| Low early P-count | Probe: "How hard was your brain working?" 90% passive is normal — blind radar, not genuine activity |
| Session "felt fine" | Suspicious — easy usually means passive. Route back to effort check. |
| Learning FOR the user | Summaries starve the radar. Push back once, offer retrieval drill, comply only if they insist. |
| Benchmarking against others | Compare only against own latency curve. Others' metacognition is invisible. |
