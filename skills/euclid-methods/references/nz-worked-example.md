# Worked Example: NZ Trip Preparedness Analysis

This is a real incident from June 14, 2026 — a single point of failure (lost phone) cascading into $380 in costs and a life-threatening drive. The analysis applies all six Euclidean habits to a concrete failure.

**Context:** User was rushing to catch a 1:00 AM bus from Canberra to Sydney for an international flight to Auckland. Phone slipped out in an Uber. No device redundancy. Exhausted overnight drive. Work unfinished. 

**Six habits applied to this incident:**

## Habit 1 — Define Terms
"Preparedness" decomposed into five domains: item security, device redundancy, financial redundancy, time margin, task completion. Preparedness = the state where **no single failure can cascade**.

## Habit 2 — State Assumptions
Seven implicit assumptions exposed:
1. "I can remember all critical items while rushing" → False
2. "My phone won't fall out in an Uber" → False
3. "If I lose my phone, I can use my watch" → False (not paired)
4. "Tablet banking works because phone banking works" → False (different verification)
5. "I can drive 300km exhausted" → False — life-threatening
6. "I can appear to finish work" → False
7. "Rushing saves time" → False — cost $380

## Habit 3 — Construct the Chain
Untidiness → Rushing → Phone slip → No backup → Stranded → $380 chase → Exhausted drive → Unfinished work in Auckland. Every step earned from the previous one.

## Habit 4 — Cite the Rule
The violated rule: "Every critical item must have a designated home. Every critical function must have a tested backup. Every departure must have a time buffer."

## Habit 5 — Dependency Map
Five corrective systems: Item Discipline, Device Redundancy, Financial Redundancy, Time Margin, Task Completion. Each with a CHECK step requiring no thinking — just execution.

## Habit 6 — Assume the Opposite
The claim "I can wing it" was tested by living it. The assumption destroyed itself: financial loss + physical danger + professional compromise. Proof complete.
